# Terminal Movies Learn

Небольшая терминальная лаборатория для изучения собственного вкуса в сериалах.
Проект собирает личные оценки, сравнивает их с рейтингами Кинопоиска и IMDb,
учится подбирать веса признаков и показывает, какие параметры сильнее всего
влияют на прогноз.

Главная идея простая: вместо безличного общего рейтинга получить модель,
которая постепенно начинает понимать, что интересно именно вам.

## Что уже умеет проект

- хранить оценки сериалов и автоматически делать резервные копии;
- редактировать датасет через обычную Excel-таблицу;
- рассчитывать прогноз личной оценки по рейтингам, популярности и тегам;
- подбирать веса несколькими способами: перебором, случайным поиском и
  последовательным обучением с разными шагами;
- сравнивать модель с базовым рейтингом Кинопоиска;
- показывать влияние отдельных признаков;
- проверять модель методом leave-one-out;
- добавлять и удалять теги через терминальное меню без ручной правки кода.

## Метрики

На главном экране выводятся две ошибки:

- `MAE` - средняя абсолютная ошибка обученной модели;
- `KP_MAE` - средняя абсолютная ошибка рейтинга Кинопоиска относительно ваших
  оценок.

Обе метрики отображаются в процентах от десятибалльной шкалы. Чем меньше
значение, тем точнее прогноз.

## Быстрый старт

Нужен Python 3 и библиотека `openpyxl`.

```powershell
pip install -r requirements.txt
python main.py
```

При первом запуске программа сама создаст рабочие JSON-файлы и каталоги.

## Обычный рабочий цикл

1. Запустите `python main.py`.
2. Откройте раздел `Данные`.
3. Выберите `Открыть датасет в Excel`.
4. Отредактируйте таблицу и сохраните файл.
5. Закройте Excel и выберите `Загрузить датасет из Excel`.
6. Перейдите в раздел `Обучение` и запустите подбор весов.
7. Сравните `MAE` модели с `KP_MAE` на главном экране.

Excel-файл для редактирования создаётся по адресу:

```text
C:/TXT_FILES/movies-learn/edit_dataset.xlsx
```

Перед импортом таблицу лучше закрыть в Excel, чтобы Windows не блокировал
файл для чтения или записи.

## Теги сериалов

Теги описываются в [tags.json](tags.json). Сейчас модель использует:

| Тег | Значение |
| --- | --- |
| `has_crime` | Криминальная линия |
| `has_psychology` | Психологическая составляющая |
| `has_romantic_pursuit` | Романтическое добивание |
| `has_after_work_watch` | Можно смотреть после работы |
| `has_survival_game` | Игра на выживание |
| `has_sci_fi_anomaly` | Фантастическая аномалия |
| `has_serious_stakes_tone` | Серьёзный тон ставок |

Каждый тег бинарный: `0` или `1`.

Структуру тегов можно менять из меню:

```text
Дополнительно -> Настройка тегов
```

При добавлении нового тега существующие записи получают значение `0`.
Перед изменением структуры программа сохраняет резервные копии и просит
перезапустить приложение.

## Где лежат данные

Рабочие данные хранятся отдельно от репозитория:

| Назначение | Путь |
| --- | --- |
| Датасет | `C:/DATA/movies-learn/dataset.json` |
| Веса модели | `C:/DATA/movies-learn/weights.json` |
| Метаданные | `C:/META/meta-movies-learn/meta_data.json` |
| Резервные копии | `C:/BACKUP/movies-learn/BACKUP/` |
| Excel для редактирования | `C:/TXT_FILES/movies-learn/edit_dataset.xlsx` |

## Структура проекта

| Файл | Отвечает за |
| --- | --- |
| `main.py` | терминальные меню и сценарии работы |
| `model.py` | прогнозы, ошибки и подбор весов |
| `storage.py` | JSON, CSV, резервные копии и совместимость старых данных |
| `excel_work.py` | экспорт и импорт Excel |
| `tags.json` | активная схема тегов |
| `tags_work.py` | изменение структуры тегов через терминал |
| `scheme.py` | схема полей и валидаторы |
| `format_score.py` | расчёт производных признаков |
| `test.py` | основные проверки проекта |
