# Terminal Movies Learn

Консольный проект для личной модели вкуса по фильмам и сериалам.

Программа хранит ваш датасет оценок, подтягивает числовые данные из API, поддерживает вайб-теги и жанровую разметку, обучает модель и отдельно ведёт общий пул кандидатов, которых ещё нет в основном датасете.

## Что умеет проект

- хранить `dataset`, `meta` и `weights` в JSON;
- добавлять записи вручную с подсказками из API;
- безопасно переименовывать запись сразу в `dataset` и `meta`;
- выгружать и загружать датасет через Excel;
- хранить и настраивать вайб-теги;
- загружать жанровую разметку из API с подтверждением;
- динамически расширять жанровые признаки по новым жанрам;
- обучать модель несколькими режимами;
- считать `MAE` и `KP_MAE`;
- вести общий пул кандидатов;
- собирать кандидатов по сохранённым критериям;
- показывать кандидатов по конкретному набору критериев;
- удалять целый пулл и все его объекты из общего пула;
- искать подозрительные дубли в общем пуле;
- считать топ `N` из общего пула по предикту без вайб-тегов;
- логировать все API-запросы в файл;
- делать backup и восстанавливать датасет.

### Переименование записи

Для исправления опечатки в названии лучше использовать отдельный пункт `Переименовать запись`, а не правку `title` через Excel.

Этот режим:

- ищет запись по старому названию;
- проверяет, что новое название валидно;
- не даёт создать дубль;
- меняет ключ и `main_info["title"]` одновременно в `dataset` и `meta`.

## Общий пул кандидатов

Общий пул — единственный источник правды для кандидатных сериалов.

Все новые подборки:

- не создают отдельные независимые хранилища;
- сливаются в один общий `candidate_pool.json`;
- при просмотре фильтруются по `criteria_name`;
- автоматически очищаются от сериалов, которые уже есть в основном датасете.

Если вы добавили сериал в основной датасет, он автоматически удаляется из общего пула кандидатов.

## Критерии подбора кандидатов

Критерии сохраняются по имени в отдельный JSON.

Поддерживаемые поля:

- страна
- количество кандидатов
- минимальный `KP`
- минимальный `IMDb`
- минимум голосов `KP`
- минимум голосов `IMDb`
- минимальный год
- максимальный год
- жанры

Все необязательные поля показываются как:

`[не важно]`

Это значит: фильтр по этому полю не применяется.

### Жанры в критериях

Жанры выбираются не текстом, а по номерам из списка.

Пример:

```text
1. Драма
2. Комедия
3. Криминал
```

Ввод:

`1 2 3`

Это убирает ошибки вроде опечаток в жанрах.

## Просмотр пулов

Можно выбрать сохранённый набор критериев и посмотреть всех кандидатов этого набора в консоли.

По каждому кандидату выводятся:

- название
- год
- `KP`
- `IMDb`
- `KP votes`
- жанры
- короткое описание

## Топ из общего пула

Отдельный пункт `Собрать топ из общего пула`:

- берёт всех кандидатов из общего пула;
- считает предикт на текущих весах модели;
- принудительно ставит все вайб-теги в `0`;
- использует только `bias`, числовые признаки и жанры;
- выводит топ `N` в формате:

`Название (год): предикт`

## Удаление пулла

Пункт `Удалить пулл`:

- удаляет выбранный набор критериев из `candidate_criteria.json`;
- удаляет все его объекты из общего `candidate_pool.json`;
- не затрагивает остальные пуллы и остальные записи общего пула.

## Подозрительные дубли

Пункт `Показать подозрительные дубли`:

- ищет пары очень похожих названий с одинаковым годом;
- ничего не удаляет;
- помогает руками проверить, не попал ли один и тот же сериал в пул под разными вариантами названия.

Сравнение учитывает:

- нормализацию регистра;
- `ё/е`;
- пунктуацию;
- пробелы;
- небольшие опечатки;
- похожесть строк.

## Структура данных

Каждая запись датасета содержит:

- `main_info`: `title`, `user_score`, `year`
- `raw_scores`: `kp_score`, `kp_votes`, `imdb_score`, `imdb_votes`
- `computed_scores`: вычисляемые числовые признаки
- `tags_vibe`: пользовательские вайб-теги
- `genre`: жанровая разметка

Признаки модели включают:

- `bias`
- числовые признаки из рейтингов и голосов
- все активные вайб-теги
- все активные жанровые признаки

## Жанры

Жанры не зашиты фиксированным списком.

Сценарий такой:

1. программа получает жанры сериала из API;
2. пользователь подтверждает совпадение или вручную правит список;
3. новые жанры автоматически добавляются в каталог жанровых признаков;
4. жанры начинают участвовать в датасете, весах и прогнозе.

Имена жанровых признаков приводятся к стилю `has_*`, например:

- `драма -> has_drama`
- `боевик -> has_action`
- `мелодрама -> has_melodrama`
- `фантастика -> has_fantasy`

## Добавление новой записи

При добавлении записи программа:

1. ищет сериал через API;
2. показывает название, год, страну и краткое описание;
3. показывает жанры из API;
4. предлагает принять жанры или изменить их вручную;
5. подставляет найденные числовые значения;
6. даёт вручную заполнить оценку, вайб-теги и жанровую разметку.

Если API вернул новый жанр, он сразу добавляется в каталог жанровых признаков и становится доступен в текущем вводе.

## Обучение

### Эвристические режимы

- `Быстрое обучение`
- `Случайная оптимизация`
- `Многошаговый координатный поиск`
- `Гибридная оптимизация`

### Линейные режимы

Отдельный модуль `model_work/linear_regression_train.py` поддерживает:

- `Ridge`
- `Lasso`
- `ElasticNet`
- `SGDRegressor (MAE)`
- `scipy minimize (MAE)`

Особенности:

- используется текущий набор признаков, включая `bias`;
- веса после линейной регрессии не нормализуются до суммы `1`;
- новые веса сохраняются только если не ухудшают `MAE`.

## Excel

Excel-схема строится из актуальных полей:

- `main_info`
- `raw_scores`
- `tags_vibe`
- `genre`

Через Excel безопаснее менять числовые поля, теги и жанры.

Название записи лучше менять через отдельный пункт меню `Переименовать запись`, потому что `title` используется как ключ записи.

## API и лог запросов

Все API-запросы пишутся в лог:

`C:/DATA/movies-learn/api_requests.log`

В лог попадают:

- старт запроса;
- URL;
- номер попытки;
- timeout;
- `HTTP`-ошибки;
- сетевые ошибки;
- успешные ответы;
- число `docs`, если JSON разобрался.

## API и ограничения сети

Сбор кандидатов использует каталоговый запрос к Kinopoisk API.

Для устойчивости:

- есть короткая preflight-проверка доступности API;
- запросы идут небольшими страницами;
- на `403`, timeout и сетевые ошибки есть повторные попытки;
- между страницами есть пауза;
- в отчёте по сбору показывается последняя страница и текст ошибки.

Если API недоступен по сети, пул не пополняется, но уже сохранённый общий пул остаётся доступен для просмотра и ранжирования.

## Что важно помнить

- вайб-теги могут отсутствовать полностью, `tag0` больше не используется;
- жанры расширяются динамически;
- `bias` уже встроен во все режимы обучения;
- общий пул кандидатов очищается от уже просмотренных сериалов;
- топ из пула считается без вайб-тегов;
- линейный и эвристические режимы сохраняют веса только если ошибка не ухудшилась.
